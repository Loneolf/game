# Utils 包导出配置分析与建议

## 现状分析

### 项目结构
- 这是一个 monorepo 项目，utils 包位于 `packages/utils`
- 主要文件：
  - `package.json` - 配置了包信息
  - `index.ts` - 主要导出文件
  - `index.d.ts` - 手动编写的类型声明文件
  - `archive/` 目录 - 包含工具函数
  - `store/` 目录 - 包含存储相关功能

### 当前导出配置
- `package.json` 中设置了 `main: "index.ts"` 和 `types: "index.d.ts"`
- `index.ts` 直接导出了各个工具函数和子模块
- `index.d.ts` 手动编写了类型声明
- `store.d.ts` 文件不存在，但 `package.json` 中引用了它

## 建议方案

### 1. 是否应该直接导出 ts 文件

**当前状态**：直接导出 ts 文件（`main: "index.ts"`）

**建议**：
- 对于开发环境，可以直接导出 ts 文件
- 对于生产环境，建议构建后导出编译后的 js 文件
- 原因：
  - 直接导出 ts 文件需要消费方也支持 TypeScript
  - 编译后的 js 文件具有更好的兼容性
  - 可以通过构建工具优化输出文件大小

### 2. 是否需要导出类型声明文件

**当前状态**：已配置类型声明文件（`types: "index.d.ts"`）

**建议**：
- 是的，TypeScript 项目必须导出类型声明文件
- 类型声明文件对于 TypeScript 消费方至关重要
- 确保类型声明文件与实际代码保持同步

### 3. 类型声明文件应该自动生成还是手动写

**当前状态**：手动编写类型声明文件

**建议**：
- 强烈建议通过 TypeScript 编译器自动生成类型声明文件
- 原因：
  - 自动生成可以确保类型与代码完全一致
  - 减少手动维护的工作量和错误
  - 提高开发效率

## 改进方案

### 1. 添加构建配置

1. **修改 package.json**：
   - 添加构建脚本
   - 修改 main 字段指向编译后的 js 文件
   - 保持 types 字段指向自动生成的类型声明文件

2. **添加 tsconfig.json**：
   - 配置 TypeScript 编译选项
   - 启用 `declaration: true` 自动生成类型声明文件
   - 配置输出目录

### 2. 修复 store.d.ts 引用问题

- 移除 `package.json` 中不存在的 `store` 字段
- 或者为 store 模块添加类型声明文件

### 3. 优化导出结构

- 保持现有的导出方式，但确保所有模块都有对应的类型声明
- 考虑添加构建步骤，确保代码质量和类型一致性

## 具体实施步骤

1. 添加 tsconfig.json 配置文件
2. 修改 package.json 添加构建脚本
3. 运行构建命令生成类型声明文件
4. 验证类型声明文件的正确性
5. 更新包的导出配置

通过以上改进，可以确保 utils 包在 monorepo 环境中正确导出公共方法，同时提供准确的类型声明，提高代码质量和开发效率。
